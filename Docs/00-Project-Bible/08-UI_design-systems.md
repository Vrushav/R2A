# R2A – UI Design System

Version: 1.0

Status: Draft

---

# Purpose

This document defines the visual design language of the R2A platform.

The goal is to ensure consistency, accessibility, and usability across all modules.

---

# Design Philosophy

R2A is designed for rural-first healthcare.

The interface should be:

- Simple
- Friendly
- Accessible
- Professional
- Mobile-first

The design should help users complete tasks with minimum effort.

# Buttons

Primary Button

- Blue background
- White text
- Rounded corners

Secondary Button

- White background
- Blue border

Danger Button

- Red background

---

# Form Controls

Input Fields

- Rounded corners
- Large click area
- Visible focus state

Dropdowns

Checkboxes

Radio Buttons

Should follow the same spacing and styling.

---

# Cards

Every card should include:

- Title
- Description
- Clear action

Spacing should remain consistent throughout the application.

---

# Icons

Use Font Awesome during Version 1.

Future versions may migrate to custom SVG icons.

---

# Accessibility

Minimum contrast ratio

Large touch targets

Readable typography

Keyboard navigation support

Clear focus indicators

# UI Principles
